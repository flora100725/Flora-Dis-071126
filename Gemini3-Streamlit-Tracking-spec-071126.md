為了協助您打造這款專門針對台灣食藥署（TFDA）**《醫療器材來源流向資料建立及管理辦法》**法規、處理萬級以上規模資料流的 AI Agent 醫療器材追溯 Web App，我將從**法規合規性、大數據清洗、前端動態圖表、AI 核心魔法功能、Agent 互動架構**以及**稽查報告生成**等關鍵維度，為您梳理出 30 個核心開發設計問題。

### 一、 法規合規與資料對齊層面 (Regulations & Data Alignment)

1. **法規申報主體識別**：法規規定申報主體包含許可證所有人與販賣業者。系統如何透過「申報業者」代碼（如 A00013、B00047）自動勾稽其在 TFDA 登記的執照類型與角色？
2. **來源與流向雙向勾稽 (Reconciliation)**：購買資料（進貨）與流向資料（出貨）的產品序號/批號（如 `RNJ146480G2001`）經常存在時間差，系統應如何設計彈性時間視窗（Time Window）來判定兩端申報是否完全對齊？
3. **黑名單與未申報偵測**：要揪出「未依規定申報」的許可證持有者或醫院，系統是否需要預先匯入全台「應申報醫材許可證主檔」與「特約醫療院所名冊」作為對比基準（Baseline）？
4. **UDI-DI 與產品識別**：範例中 `UDI_DI`（如 `00763000956004`）對應特定型號，當申報欄位遺漏產品型號或次類別時，系統如何利用 UDI 條碼資料庫進行自動補全？
5. **保存期限與警示**：針對特定植入性醫材（如範例中的心臟節律器），法規對保存期限與有效期間有嚴格要求，系統該如何針對「已出貨但即期」或「過期未回收」的流向自動觸發法規合規警示？

### 二、 資料架構與巨量清洗層面 (Data Architecture & Cleaning)

6. **欄位異質性處理**：購買資料有「供應商」與「接收日期」，流向資料有「供應對象」與「交貨日期」，系統如何將兩套不同維度的 Schema 在記憶體中高效轉換為統一的「流通履歷鏈」？
7. **髒資料與特殊字元清洗**：範例中品名出現特殊引號（如 `“美敦力”`）或型號內含雜訊（如 `# 美敦力亞士爾...`），Python 清洗模組應採用何種正規表示式（Regex）標準化策略以防影響後續 AI 分析？
8. **萬級資料即時效能**：面對每年單一業者超過 10,000 筆、全台累計百萬級的動態紀錄，Python 後端（如 Pandas/Polars）應如何優化動態篩選器的回應速度，確保 Web UI 不延遲？
9. **退貨與剩餘數量邏輯**：購買資料中含有「退貨資訊」與「剩餘數量」，系統該如何設計庫存與流向扣減邏輯，以精準計算出每一批號醫材在各節點的「真實留存量」？
10. **地理資訊勾稽（GIS）**：結合 DHA 節點地理資料庫，當申報代碼（如 B00047）在流向檔中出現，但地址與地理座標對不上時，系統應如何提示使用者進行修正？

### 三、 5 大 Wow 視覺化圖表設計 (5 Wow Graphs Design)

11. **Wow 圖表 1：全台醫材流向動態軌跡地圖 (Dynamic Supply Chain Flow Map)**
* 使用者在前端設定「許可證號/日期區間」過濾器後，如何利用 Python (Plotly/Folium) 繪製從進口商(B)到經銷商(B)再到各醫院(A)的**動態流向地理線條拓撲圖**，並用粗細代表數量？


12. **Wow 圖表 2：來源與流向申報斷鏈網絡圖 (Declaration Disconnect & Gap Network)**
* 如何設計一個**關聯網絡圖 (Network Graph)**，清晰標示出哪些節點（如某特定醫院或盤商）有出貨紀錄卻完全沒有進貨申報（反之亦然），並用紅色亮點視覺化呈現「法規漏洞斷鏈點」？


13. **Wow 圖表 3：產品生命週期與過期風險熱圖 (Product Lifespan & Expiry Risk Heatmap)**
* 如何依據製造與有效日期，動態產出**甘特圖結合熱圖 (Heatmap)**，依據型號與批號展現全台庫存的「即期與倒數天數分佈」，讓稽查員一眼看出哪些高風險醫材即將過期且滯留在哪些醫院？


14. **Wow 圖表 4：申報時序延遲與漏報趨勢圖 (Reporting Latency & Omission Trend)**
* 購買資料有「建立日期」與「收貨日期」，兩者之差即為申報延遲。如何設計一組**多軸複合圖表**，讓使用者依業者過濾，呈現哪些業者的申報習慣嚴重落後法規規定的天數？


15. **Wow 圖表 5：高風險植入物全台病患終端留存漏斗圖 (Implants Terminal Retention Funnel)**
* 針對心臟節律器等第三等級醫材，如何建立一個**動態桑基圖 (Sankey Diagram) 或漏斗圖**，追蹤從進口總量到分支管道，最終到「未確認申報終端」的黑洞比例？



### 四、 5 大 AI 魔法功能設計 (5 Wow AI Features / AI Magics)

16. **AI Magic 1：法規合規自動審查與紅旗預警 (Automated Regulatory Compliance Red-Flagging)**
* 經由 AI 掃描萬筆資料後，自動產出「法規異常用語、異常流向、幽靈經銷商（有統編無真實基地）」的**自動化紅旗報告書**，其 Prompt 與檢驗邏輯該如何封裝？


17. **AI Magic 2：全台醫材黑市/囤積/異常流向預測 (Anomaly Flow & Anomaly Volume Detection)**
* 如何利用機器學習或 LLM 識別特定批號在短時間內於複數經銷商間「循環對調」或「突然暴增」的異常交易行為，精準預測潛在的囤積或非法洗貨風險？


18. **AI Magic 3：一鍵生成 TFDA 標準稽查公文與報告書 (One-Click TFDA Audit Report Generator)**
* 當系統撈出未申報名單後，如何讓 AI 一鍵調用台灣公文格式，自動帶入違規事實（違反管理辦法第幾條）、漏報統計，直接產出**格式完美的執法公文草卷**？


19. **AI Magic 4：智慧欄位對齊與模糊勾稽引擎 (Smart Schema Alignment & Fuzzy Matching)**
* 當不同通路端上傳的 CSV 欄位名稱極度混亂（例如：`許可證號` vs `證書字號` vs `License_No`），AI 如何在背景自動識別語意、對齊欄位並自動完成清洗對照？


20. **AI Magic 5：智慧型互動式數據偵探探針 (AI Data Detective Conversation)**
* 使用者選定特定篩選條件後，AI 能否「主動探測」並彈出提示（例如：「*提示：我發現您篩選的 W3DR01 型號，在台中榮總與美敦力之間的申報數量有 1 筆對不上的誤差，是否需要我為您調出該批號的履歷？*」）？



### 五、 Agent 互動與 Prompt 機制 (Agentic Interaction & Prompting)

21. **Prompt 介面權限與角色分工**：由於涉及敏感的醫療器材流通資料，當「TFDA 稽查員」與「醫材廠商」輸入相同的 Prompt 時，AI Agent 應如何動態過濾並限制其可調用及處理的資料範圍？
22. **多步驟 Agent 任務拆解 (Chain of Thought)**：當使用者 Prompt 輸入：「*幫我查出今年度所有節律器申報不一致的醫院名單並分析原因*」，Agent 該如何將其拆解為「資料撈取 -> 邏輯比對 -> 統計繪圖 -> 語意解讀」的自動化管道（Pipeline）？
23. **Code Interpreter 安全隔離性**：系統允許 AI 執行 Python 程式碼來處理資料與繪圖，如何在高併發（High Concurrency）的環境下確保執行環境（如 Docker Sandbox）的安全，防止惡意程式碼注入？
24. **動態過濾參數與 Prompt 融合**：當使用者在 UI 勾選了「日期：2026Q1、醫材次類別：E.3610」，隨後又在對話框輸入 Prompt，系統如何將 UI 篩選狀態（State）完美轉譯為 AI 的 System Prompt 上下文？
25. **AI 繪圖指令中文化與錯漏修正**：使用者可能用極度口語的中文要求改圖（如「*把剛剛地圖上台北的部分放大，線條改鮮紅色*」），後端 Python 代碼生成器如何精準對應 Matplotlib/Plotly 的參數並正確顯示中文字型？

### 六、 報告生成與稽查應用層面 (FDA Reporting & Audit Application)

26. **幽靈鏈結追蹤機制**：如範例中購買資料顯示 `A00013` (台大) 向 `C00306` 購買，但地理資料庫與流向資料中完全沒有 `C00306` 的申報紀錄，系統應如何將此類「單邊申報幽靈供應商」在報告中獨立歸類？

28. **歷史申報快照與版本控制**：廠商可能會修正已上傳的申報資料（例如補上產品序號），系統如何記錄「歷史版本」，好讓 TFDA 追溯廠商是否存在「事後規避稽查的串改行為」？
29. **批號級別召回模擬 (Lot-Level Recall Simulation)**：若發生重大醫材不良事件（例如特定批號 `RNE644338S` 發生重大瑕疵），系統能否一鍵啟動「AI 模擬召回」，在 3 秒內精準定位該批號目前在全台哪些醫院、被哪些盤商經手、剩餘多少數量？
30. **匯出格式與電子簽章整合**：產出的 PDF 綜合報告，除了文字與 5 大 Wow 圖表外，是否需要預留對接台灣政府公文 G2B 系統的 XML 格式，或是預留動態浮水印與電子簽章的欄位？
