1. 系統架構與設計哲學 (System Architecture & Design Philosophy)
1.1 核心設計哲學
AURA-7 Hub v2（以下簡稱「本系統」）是專為第三類（Class III）高風險植入式醫療器材（如主動式心臟節律器、心臟去顫器、人工心臟瓣膜等）量身打造的主動式代理人監管作業系統（Agentic OS）。
在傳統的醫療器材監管體系中，數據流動往往面臨「滯後性」與「孤島化」兩大痛點：
數據申報滯後：進口商、經銷商與醫療機構之間的帳籍對帳通常依賴季報或月報，導致異常流向（如出庫未登、幽靈帳籍）無法在第一時間被攔截。
法規判定孤島：當發生重大的冷鏈失溫或產品缺陷時，法規裁決需要召集多方專家開會，動輒耗時數週，對患者生命安全構成嚴重威脅。
本系統的設計哲學在於**「將靜態的監管日誌轉化為動態的主動防禦網絡」。本系統不只是一個數據庫，而是一個整合了大語言模型（Gemini-3.1-Flash-Lite）**、圖論算法（Graph Algorithms）、**地理信息系統（GIS）與多智能體共識網絡（Autonomous Multi-Agent Consensus Swarm）**的數位孿生監管沙盒。其核心架構以「數據實時治理」、「自動合規重整」與「主動安全預警」為三大支柱，確保每一枚植入患者體內的心臟節律器皆具備絕對的可追溯性與物理安全性。
1.2 軟體架構組件
系統採用全棧架構，前端基於 React 19 + TypeScript + Vite + Tailwind CSS，後端採用 Node.js (Express) + TypeScript (tsx) 驅動。所有關鍵的生成式人工智慧（GenAI）與物理模擬運算均在伺服器端完成，確保 API 金鑰之安全不外洩，並提供穩健的併發處理能力。

2. 數據流與安全機制 (Data Flow & Security Mechanics)
2.1 進銷存雙軌數據流 (Dual-Track Ledger Synchronization)
本系統的核心數據庫分為兩大主軌：
進貨申報帳籍 (Purchase Ledger / Inbound)：記錄進口商（如美敦力台灣分公司）將醫療器材運抵台灣，報關、入庫並於中央法規節點登記的詳細資訊。
出貨與配送帳籍 (Distribution Ledger / Outbound)：記錄經銷商與物流中心將器材運抵各大醫療機構（如台大醫院、榮總、長庚）的配送數據與臨床植入登記。
這兩軌數據通過唯一的 UDI（Unique Device Identification，醫療器材單一識別碼） 體系進行關聯。一個完整的 UDI 碼由以下兩部分組成：
UDI-DI (Device Identifier，產品識別碼)：通常為 14 位的 GTIN 條碼，用於識別特定品牌與型號。
UDI-PI (Production Identifier，生產識別碼)：包含批號（Lot Number）、序列號（Serial Number）與有效期限（Expiry Date）。
本系統的數據標準化引擎會實時攔截這兩條軌道的數據，進行動態關聯分析（Dynamic Joins），檢測是否存在不一致性。

JSON 自適應器：解析標準的 JSON 陣列，若輸入為單一 JSON 對象，會自動將其包裝為單元素陣列以保證向下流動的一致性。

3.3 審計數據自動補正精靈 (Audit Data Auto-Correction Wizard)
在申報數據中，最致命的缺陷之一是**「生產日期（Manufacturing Date）」或「保存期限（Shelf Life/Expiry Date）」缺失**。這可能導致醫院無法及時排查過期醫材，引發醫療事故。
本系統創立了審計數據自動補正精靈。該精靈內置了批次序列號本體論（Lot-SN Ontology Pattern）。
例如，美敦力（Medtronic）的序列號編碼具有極強的物理規律性：
前綴 RNJ14... 代表該批次於 2024年6月28日 製造，其出廠保存期限為 3 年，故預設過期日應補正為 20270628。
前綴 RNJ13... 代表該批次於 2024年1月14日 製造，預設過期日應補正為 20270114。
前綴 RNE64... 代表該批次於 2023年12月14日 製造，預設過期日應補正為 20261214。
當稽查官啟動此功能時，系統會先掃描整張報表，定位所有保存期限為空的值，隨後調用 AI 代理人，比對內置的批次序列號規律，自動推理出最精確的製造與到期日期，並標註為 [AI-Corrected]，極大地減少了人工複查的時間成本。

4.1 供應鏈拓撲網絡圖 (Supply Chain Topology Network Graph)
此圖表採用了動態力導向布局算法 (Force-Directed Layout Algorithm)。
節點 (Nodes)：代表供應鏈中的不同實體（如進口商 A00013、北部經銷商 B00021、台大醫院 H00092）。
連線 (Edges)：代表產品的調撥和物流方向。
動態力學模擬：
系統使用庫侖定律（Coulomb's Law）計算節點間的斥力，防止節點重疊；同時使用胡克定律（Hooke's Law）計算連線的彈性引力，使關係緊密的節點凝聚在一起。連線的粗細與申報的產品數量成正比。若某個連線存在「出庫未登」的不合規嫌疑，該連線會被著色為高飽和度的霓虹紅色（Neon Red）並帶有流體流動動畫，提示稽查官此處存在斷鏈風險。
4.2 GIS 分佈追蹤地圖 (Geographic GIS Traceability Map)
地圖模組基於台灣真實的地理坐標系（TWD97 / 經緯度投影）。
地圖層：繪製台灣本島及外島的邊界與主要醫學中心坐標。
物流航線：利用貝茲曲線（Bezier Curves）繪製從進口商報關口岸（如基隆港、桃園機場）到各級醫院的跨縣市配送軌跡。
異常警示：若某個調撥節點在冷鏈 telemetry 監控中發生了「失溫事件」，該地圖坐標上會觸發一個帶有二級漣漪擴散特效（Ripple Effect）的紅色呼吸燈節點，稽查官可點擊該節點查看當下的物理溫度衰減曲線與承運司機的聯絡資訊。
5. 3個主動安全沙盒模擬器原理 (Autonomous Regulatory Sandbox Simulators)
5.1 司法時序悖論與數據黑洞追蹤器 (Forensic Supply Chain Temporal Graph & Chronological Paradox Visualizer)
在物理世界中，一件物品必須先被進口並登記，隨後才能被配送給醫院。然而，在不合規的申報中，經常出現以下兩種時序異常：
時序逆轉悖論 (Temporal Inversion Paradox)：

即醫院端登記的植入日期，早於進口商向 TFDA 申報該產品入庫的日期。這通常暗示著非法平行輸入（水貨）、仿冒品流入或申報數據惡意回填/造假。
數據黑洞 (Data Black Holes)：
產品在進口端有申報登記，但在系統時間超過 30 天後，既無經銷商調撥記錄，亦無醫院端接收記錄。該產品在監管視線中「憑空消失」，極有可能流入了未經登記的地下醫療市場，對患者生命構成嚴重隱患。
沙盒會自動將進銷存兩端數據構建為單向有向無環圖 (DAG)，利用拓撲排序（Topological Sort）檢測圖中是否存在「環路（Cycles）」或「逆向弧（Reverse Edges）」，一旦發現即將其標記為時序悖論，並引導 AI 進行成因分析。

當原廠發布某特定批次的心臟節律器存在「電池電極焊點微裂紋（Solder Joint Micro-cracks）」或「密封絕緣老化」的緊急回收通知時，監管部門必須立即回答：「這些有缺陷的醫材現在在哪？已經植入多少人體內？」

5.3 國安級醫療供應鏈壓力測試沙盒 (National-Security Healthcare Supply Chain Stress-Tester)
此沙盒用於模擬極端外部衝擊下，台灣整體高風險醫療器材供應鏈的抗震彈性（Resilience Margin）：
模擬場景一：海關關口無限期關閉（Border Suspension）：
模擬進口通道完全切斷。系統會依據當前的每日臨床消耗速率（Consumption Velocity），精確計算出各大醫院心臟節律器的安全庫存耗盡天數（Stockout Runway）。
模擬場景二：冷鏈物流中斷（Cold Chain Collapse）：
模擬夏季高溫下，物流車隊製冷故障。系統根據冷鏈衰減公式，預估受波及醫材的報廢率。
模擬場景三：原廠產線無限期停工（Factory Shut down）：
評估替代型號的承接能力與市場缺口。
